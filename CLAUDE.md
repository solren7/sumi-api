# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
make run        # go run main.go (starts root CLI)
make api        # go run main.go api (starts HTTP server)
make build      # build binary to bin/server
make generate   # run sqlc generate (regenerates internal/repository/dbgen/)
```

Run tests: `go test ./...`
Run single test: `go test ./internal/services/... -run TestName`
Lint: `golangci-lint run` (if installed)

## Architecture

This is a Go REST API using [Fiber v3](https://github.com/gofiber/fiber) for a personal finance/bill tracking app.

### Request Lifecycle

```
main.go → cmd/ (Cobra) → internal/apps/app.go → StartAPIServer()
  ↓
Fiber app + Middleware stack (logger, CORS, error handler, panic recovery)
  ↓
internal/apps/router.go → handlers/ → services/ → repository/dbgen/ (sqlc) → PostgreSQL
```

### Layer Responsibilities

| Layer | Path | Role |
|-------|------|------|
| Config | `config/config.go` | Env var parsing via `caarlos0/env` |
| Database | `internal/database/` | pgx connection pool + Redis client |
| Repository | `internal/repository/dbgen/` | **Generated by sqlc — do not edit manually** |
| Services | `internal/services/` | Business logic; aggregated in `service.go` |
| Handlers | `internal/handlers/` | Fiber context parsing; aggregated in `handler.go` |
| Middleware | `middleware/` | JWT auth, error formatting, context helpers |
| Utilities | `pkg/` | `logx/` (zerolog wrapper), `errorx/` (typed errors), `utils/` (JWT gen/parse) |

### Database

- PostgreSQL via pgx/v5; Redis via go-redis/v9
- Schema: `sqlc/schema.sql` | Queries: `sqlc/queries.sql`
- After modifying SQL files, run `make generate` to update `internal/repository/dbgen/`
- sqlc overrides: `pgtype.UUID` → `uuid.UUID`, numeric → `decimal.Decimal`, timestamps → `time.Time`

### Auth

- JWT (HS256, 72h expiry) stored as Bearer token
- `middleware/auth.go` validates token and stores `user_id` + `email` in Fiber `Locals`
- Passwords hashed with bcrypt

### Error Handling

- Use `pkg/errorx` to create errors with HTTP status codes: `errorx.New(http.StatusBadRequest, "message")`
- Global error handler in `middleware/error.go` returns `{"success": false, "code": N, "message": "..."}`
- Logging via `pkg/logx`: `logx.WithError(err).WithField("k", "v").Error("msg")`

### Routes

```
POST /api/auth/check-email
POST /api/auth/register
POST /api/auth/login
POST /api/bills/           (auth required)
GET  /api/bills/           (auth required)
PUT  /api/bills/:id        (auth required)
DELETE /api/bills/:id      (auth required)
GET  /api/stats/home       (auth required)
```
